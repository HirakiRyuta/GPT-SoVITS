# CLAUDE.md — GPT-SoVITS

Claude Code がこのプロジェクトで作業するときの行動指針。

---

## セッション終了時のログ保存

**Claude Code は各セッション終了前に、glis-hq のログ保存スクリプトで作業ログを保存すること。**

```bash
bash /home/infoglis/glis-hq/scripts/ai-log-capture.sh "GPT-SoVITS" "<作業概要>" "<改善内容>" "<使用ツール>" "<学び>"
```

ログは `/home/infoglis/glis-hq/logs/ai-improvement/` に保存される。
note-automation の B4（AI改善ログ）カテゴリの記事ネタとして活用される。

### 機密情報の扱い【必須】

ログには以下の機密情報を**含めないこと**。

- プロジェクト固有のコードネーム・プロダクト名（例 — 社内プロダクト名はそのまま書かない）
- プロンプトの具体的な内容・設計の詳細
- クライアント名・案件名
- API キー・認証情報

ログには「何を改善したか」「どんな手法を使ったか」の汎用的な記述のみ残す。具体的なプロダクト名は「社内の音声合成システム」などに置き換える。
